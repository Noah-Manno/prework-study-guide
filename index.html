HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for a webpage. This element is required and is important for Search Engine Optimization.
  • The style element defines the CSS styles associated with a website.
  • The link element connects an external resource for the HTML document.
  • The meta element defines metadase such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTMLS document.
  • h1, h2, h3, h4, h5, and h6 are elements which represent the level of heading a given text block receives
  • p: represents a paragraph or block of text
  • ul: an unordered list
  • li: elements in the list
  • ol: an ordered list
  • img: to create and house images
  • src attribute defines the location, or source, of the image file.
  • alt attribute contains a text string that describes the appearance and functionality of the image.
  • br: a line of empty space or line break
  • a: an anchor creates links to the same webpage or other webpage
  • The href attribute points to the URL for the link

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • External style sheets can be used to keep your CSS code seperate from your other code.
  • External style sheets can be linked to HTML by using the link element and defining the relationship of the document and the document link next to href.
  • Adding an astrix before a line of CSS code means it will apply to every element.
  • You can add a comma in between two elements which you want to have that style.
  • You can also define a class in CSS code to make it so you can apply it to specific sections of code instead of every element of that type.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone paste copied SSH value: copies the repo into your depository

  • Github Workflow:

    1. Navigate to the repo you want to work on.
    2. Utilize git status to see which branch you are currently in.
    3. To add a branch where we can save our work-in progress we will create a new feature branch: git checkout -b name of branch.
    4. Utilize git status once again to see if you are on the new branch and not the main branch
    5. Open it up in VS Code utilizing the shortcut command
    6. Make changes
    7. To Add all changes: git add -A
    8. Commit your changes: git commit -m name of commit
    9. You may have to pull down any changes from the base branch: git pull origin main
    10. Make your push request: git push origin name of branch

  • Github Terms:

    • Repository: A folder for your project; Contains all the files and revisions.
    • Branch: Workspace where you can make changes that do not affect the live site.
    • Markdown File: The way content files are formatted in Github
      • A way to write for the web using plain text; easily converts to HTML
      • Markdown files use .md file extension
    • Commit Changes: Saving a new version of a file you are working on. Whenever you add, edit, or delete a file, you’re making a commit. Adds changes to your branch.
    • Pull Request: Asking for changes on a branch to merge into another branch; way to discuss and review your work with others.
    • Merge: What happens when a PR is approved. Changes are “pulled in” or merged into main branch. Changes can publish/deploy to the live site once this happens.
    • Issue
      • Way to report new tasks or content fixes
      • Track progress on project board
      • Open an issue for a new task and close once done
